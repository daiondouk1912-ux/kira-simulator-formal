# KirA 外構工事費用 概算シミュレーター v13.8

## 更新目的

v13.8 は、v13.7で正常動作確認できた流れを維持しながら、次の改善判断に使うための行動分析データを増やす安定優先アップデートです。

お客さま側の画面変更は最小限にし、マスター側に届く通知へ以下の分析情報を追加します。

- 滞在時間
- 入力項目数
- 問い合わせ（質問）ボタン押下通知の明確化
- GA4統計イベントへ滞在秒数・入力項目数を追加

## v13.8で入れた内容

### 1. 滞在時間の取得

シミュレーター開始から各操作までの時間を計算し、結果表示通知や保存・質問・感想などの通知に追加します。

表示例：

```text
・滞在時間: 3分42秒
```

分析例：

- 30秒程度：軽く確認、冷やかし、途中確認の可能性
- 3〜5分：内容を見ながら検討している可能性
- 長め＋保存：比較・家族相談・本気度高めの可能性

### 2. 入力項目数の追加

選択された工事項目数を通知に追加します。

表示例：

```text
・入力項目数: 2項目
```

分析例：

- 1項目：単体工事の費用確認
- 2〜3項目：複合工事の検討
- 多数項目：大きめの外構計画、または偵察・テストの可能性

### 3. 問い合わせ（質問）ボタン押下通知の明確化

結果画面の「この概算について質問する」が押された時の通知タイトルを、問い合わせ寄りに分かりやすくしました。

通知タイトル：

```text
問い合わせ（質問）ボタンが押されました
```

実質的な問い合わせ前コンバージョンとして見られるようにしています。

### 4. GA4統計イベントの情報追加

v13.7で入れたGA4イベント土台に、以下を追加しました。

- input_item_count
- elapsed_seconds

GA4測定IDを入れると、将来的に以下のような分析につなげられます。

- 結果表示までの平均時間
- 保存ボタンまで進んだ人の滞在時間
- 入力項目数別の保存率・問い合わせ率

### 5. 既存内容も維持

以下は引き続き含んでいます。

- いわき市周辺の外構工事費用向け文言
- LINE保存ボタン
- 質問ボタン
- 感想欄・感想通知
- 通知の改行表示
- 人工芝の表現調整
- 大面積レンジ調整
- 二重計算注意
- 注意書きの赤系強調
- 入力欄の見やすさ維持

## デプロイ後の確認項目

1. トップ画面が表示されるか
2. 工事項目選択 → 入力 → 結果表示まで進めるか
3. 結果表示通知に「入力項目数」「滞在時間」が出るか
4. LINE保存ボタンでLINEが開き、保存通知にも「入力項目数」「滞在時間」が出るか
5. 「この概算について質問する」ボタンでLINEが開き、問い合わせ（質問）ボタン通知が届くか
6. 感想送信通知にも「入力項目数」「滞在時間」が出るか
7. 既存の開始通知、項目選択通知、入力完了通知が届くか
8. URL末尾に `?debugStats=1` を付けて操作した時、ブラウザコンソールに `[KirA stats]` が出るか

## 注意

publicPriceMaster.js は公開用の概算レンジだけを管理するファイルです。原価・人工原価・利益率などの内部情報は絶対に入れないでください。

GA4測定IDはまだ空欄です。統計を本格的に見る場合は、HP側・GA側の準備と合わせて測定IDを設定してください。
