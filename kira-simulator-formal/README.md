# KirA 外構工事費用 概算シミュレーター v13.7

## 更新目的

v13.7 は、機能追加を増やしすぎず、今後の判断材料を取れるようにする安定優先アップデートです。

目的は以下です。

- 「いわき市 外構工事 費用」などの検索導線に合わせた文言へ調整
- 結果画面の保存・質問導線を維持しつつ、説明を少し整理
- GA4で統計イベントを取れる土台を追加
- LINE通知の項目・入力内容を改行表示にして見やすく改善
- v13.1〜v13.6までの修正内容を含んだ最新版として運用

## v13.7で入れた内容

### 1. 費用検索向けの文言調整

index.html とアプリ内の文言を、以下の方向へ寄せました。

- いわき市周辺の外構工事費用
- 相談前の費用確認
- 土間コンクリート、フェンス、人工芝、防草対策などの費用目安
- LINEに送って保存できること

### 2. 結果画面の導線整理

結果画面では、以下の流れを維持しています。

1. この概算をLINEに送って保存する
2. この概算について質問する
3. その他の工事も入力する
4. もう一度はじめから入力する

「保存」と「質問」を主導線として残し、その他・やり直しは弱めのボタン扱いにしています。

### 3. 統計イベントの土台追加

GA4の測定IDを入れると、以下のイベントを記録できる土台を追加しました。

- sim_start
- sim_selection_complete
- sim_input_complete
- sim_result_view
- sim_line_save_click
- sim_consult_click
- sim_feedback_submit
- sim_other_work_add_click
- sim_restart_click

index.html 内の以下に GA4 測定IDを入れると有効になります。

```html
window.KIRA_GA_MEASUREMENT_ID = "";
```

例：

```html
window.KIRA_GA_MEASUREMENT_ID = "G-XXXXXXXXXX";
```

空欄のままでも、シミュレーター自体は通常通り動作します。

### 4. LINE通知の見やすさ改善

結果通知・保存ボタン通知・質問ボタン通知などで、項目や入力内容を「、」区切りではなく、改行リストで見やすくしました。

### 5. 既存内容も維持

以下は引き続き含んでいます。

- 人工芝の表現調整
- 感想欄追加
- 感想通知
- 大面積レンジ調整
- 二重計算注意
- 注意書きの赤系強調
- 入力欄の見やすさ維持
- LINE保存ボタン
- 質問ボタン文言

## デプロイ後の確認項目

1. トップ画面が「いわき市周辺の外構工事費用」寄りの文言になっているか
2. 人工芝の目安選択で「広めの庭・ドッグランなど」が出るか
3. 任意の項目で結果表示まで進めるか
4. 合計欄の赤系注意書きが出るか
5. 「この概算をLINEに送って保存する」ボタンが出るか
6. LINE保存ボタンでLINEに概算文章が入るか
7. 「この概算について質問する」ボタンが出るか
8. 感想欄から感想通知が届くか
9. 結果通知の「項目」「入力内容」が改行で見やすくなっているか
10. URL末尾に `?debugStats=1` を付けて操作した時、ブラウザコンソールに `[KirA stats]` が出るか

## 注意

publicPriceMaster.js は公開用の概算レンジだけを管理するファイルです。
原価・人工原価・利益率などの内部情報は絶対に入れないでください。

GA4測定IDはまだ空欄です。統計を本格的に見る場合は、HP側・GA側の準備と合わせて測定IDを設定してください。
